# Capstone Projects

The capstone projects turn the book's concepts into complete systems.

They are intentionally placed after the four main volumes because they require ideas from the whole book:

* core syntax
* data structures
* functions
* files
* modules
* packaging
* testing
* debugging
* architecture
* APIs
* concurrency
* persistence
* internals
* deployment thinking

Each capstone is written as a project chapter.

The goal is not to copy code blindly.

The goal is to build, test, revise, and understand the system.

The order matters.

The early projects focus on local programs, files, command-line interfaces, and data modeling.

The middle projects introduce APIs, persistence, networking, background work, and framework design.

The later projects connect Python internals, event loops, interpreters, distributed systems, and production engineering.

Each project should leave the reader with working software and a sharper mental model.
